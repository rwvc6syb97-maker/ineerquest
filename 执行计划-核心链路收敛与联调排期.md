## 三、问题清单（按优先级）

### P0（阻塞发布）
- **P0-1 前端"失败即 mock fallback"系统性掩盖契约风险**
  - 现象：`usePayment / useCoaching / usePrivacy / useCareerPlan / useSkillGap / useLearningResources / useFavorites / FullReportPage 职业匹配` 等在 catch 中无条件回退 mock，无环境开关。
  - 危害：与此前登录 BizCode 错位同类——真实接口契约不符时被静默掩盖，用户看到假数据，问题在生产暴露。
  - 要求：联调阶段全部去除无条件 mock fallback，失败改为错误态呈现（对齐 useReport/useMembership 已完成的做法）。

### P1（发布前应完成）
- **P1-1 测评链路本地兜底未拆除**：IntroPage/QuizPage/GeneratingPage 本地临时 recordId 与本地推导结果，需切真实 records 接口。
- **P1-2 后台 mock 登录兜底**：`adminAuth.store` 固定账号 mock 登录需在生产关闭，强制走后端。
- **P1-3 AI 对话真实 LLM 通道**：配置 LLM Key（OxyGent/OpenAI 兼容），验证真实回复替代 mock 输出。

### P2（可迭代）
- **P2-1 shared 其它错误码对齐**：`packages/shared` 的 TOKEN_INVALID / CommonCode 仍旧值，统一对齐后端契约 v2.0。
- **P2-2 产品分析报告修订**：AI/辅导状态由"Mock 兜底"更正为"后端已实现，缺外部依赖时降级"。
- **P2-3 外部依赖接入**：Redis（时段锁/延迟队列）、MQ、ClickHouse、OSS 海报、真实支付/短信/邮件通道。

## 四、后续执行需求与验收标准

### 需求 R1：去除前端无条件 mock fallback（P0）
- 范围：上述 P0-1 列举的全部 hooks 与页面。
- 规则：接口失败 → 抛真实 ApiError → 页面呈现错误态 + 重试；禁止用假数据覆盖。
- 验收：断网/后端返回 4xxx 时页面显示错误态而非假数据；正常时展示真实后端数据；build 通过。

### 需求 R2：核心链路端到端真实联调（P0/P1）
- 范围：测评→报告→职业推荐→路线图→技能差距→学习资源→成长计划→辅导。
- 验收：每个环节前端展示的数据来源为后端真实响应，字段与契约 v2.0/v2.1 完全一致；无 mock 数据出现。

### 需求 R3：AI 对话真实通道验收（P1）
- 验收：配置 LLM Key 后，对话返回非 mock 文案；超时/失败有友好提示且保留输入。

### 需求 R4：生产安全收敛（P1）
- 验收：后台 mock 登录在生产关闭；敏感信息不泄露；RequireAuth/RequirePaid 守卫生效。

### 需求 R5：契约与文档对齐（P2）
- 验收：shared 错误码对齐后端；产品分析报告状态描述修订完成。
