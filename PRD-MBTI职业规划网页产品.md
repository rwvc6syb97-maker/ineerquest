| 项目 | 内容 |
|------|------|
| **产品名称** | TypeCareer（职型·Jung） |
| **文档版本** | V2.0（需求确认修订版） |
| **状态** | 待评审 |
| **编写人** | 产品经理 / 邓易丰（Alan） |
| **目标用户** | 首期：应届毕业生/职场新人（0-2年）+ 在校大学生；后续拓展：全年龄段 |
| **产品定位** | 以 MBTI 为入口，融合能力/兴趣/价值观维度，输出综合"职业画像"，并落点到具体岗位 + 可执行学习路径的轻量级网页工具 |

## 一、问题背景与用户洞察

### 1.1 核心痛点
- 应届生/职场新人选择第一份工作时信息不对称，迷茫度高
- 市面 MBTI 测试多但做完只给一段描述，无后续可执行内容
- 传统职业规划咨询周期长/成本高/覆盖不到大部分学生群体
- **关键缺失**：缺乏一个"低门槛 → 快速出方向 → 有具体路径"的闭环工具

### 1.2 用户画像

**首期用户（P0）：**
| 标签 | 描述 |
|------|------|
| 大二/大三/大四学生 | 正在做实习，对"选哪个方向"有模糊认知，需要参考 |
| 应届毕业生 | 拿不到满意 offer 或不确定方向 |
| 职场新人（0-2年） | 当前岗位不满意，想明确"到底该往哪走" |
| 低行动力群体 | 知道该努力但没方向，需要"看不见的手"推一把 |

**后续用户（P1-P2）：**
- 职场中坚（3-10年）—— 转行/副业/高阶规划
- 家长/教育者 —— 帮孩子做参考
- HR/培训 —— 选人参考

### 1.3 用户场景示例

> **场景A：**
> 大三学生小张，刷到了MBTI测试，做完发现自己是个 ENFP。他查了一下"ENFP适合做什么"，结果五花八门。他需要的不只是"适合做XX"，而是"做这个方向要怎么入手"。

> **场景B：**
> 应届生小李拿了一个 offer，但不确定是不是"适合自己"。他想找点参考依据，但不只是看性格，更想看"这岗位综合来看我能不能做/喜不喜欢/值不值得做"。

## 二、产品定位与核心价值

### 2.1 一句话定位

> **"做一次测试，不是给你一个标签，是给你一条路。"**

### 2.2 核心价值主张

| 维度 | 说明 |
|------|------|
| **低门槛** | 不登录/不填表/不付费，5-8分钟出结果 |
| **有深度** | 不只说性格，说"能力匹配度 + 兴趣匹配 + 价值观吻合" |
| **可落地** | 给的不是一句话，而是"这个岗位怎么入门"的3步路径 |
| **可分享** | 结果可以生成截图/卡片，社交传播 |

### 2.3 产品架构（三层）

```
┌─────────────────────────────────────┐
│              入口层                    │
│  MBTI 测试（4维度，8-20题）           │
│                                         │
│               ↓                        │
│          画像生成层                    │
│  人格类型 → 综合职业画像               │
│  （性格+能力倾向+兴趣+价值观）         │
│                                         │
│               ↓                        │
│           行动落地层                   │
│  推荐岗位 → 技能树 → 学习路径          │
│  → 可执行的3步行动计划                 │
└─────────────────────────────────────┘
```

## 四、功能模块详情

### 4.1 模块A：MBTI人格测试（入口）

#### 4.1.1 设计原则
- **时长控制**：核心8题（≥2分钟出结果），可选20题完整版
- **降低心理负担**：每页只展示1题，卡片式横滑
- **防放弃**：底部显示"已完成了 X/8 题"，进度可视化
- **题目说明**：每题配一句"通俗翻译"（如：E/I → "你是更想先说话再想，还是先想再说话？"）

#### 4.1.2 四维度×2选项

> **注意：** 题库以"通俗化"写，不用专业术语。示例：
>
> | 维度 | 题目（二选一） |
> |------|--------------|
> | E/I | 朋友聚会我能聊到最后一个 / 我感觉差不多了想先撤 |
> | S/N | 我更相信"干过才懂" / 我更相信"先想清楚" |
> | T/F | 我会优先看"这事对不对" / 我会优先看"大家开不开心" |
> | J/P | 我希望事情是安排好的 / 我不介意临时变 |

#### 4.1.3 快速入口（Barrier-free）
- 首页提供"速配"模式：点击4个描述片段中最像自己的 → 直接出结果（跳过完整测试）

### 4.2 模块B：综合职业画像（结果页——核心）

**这是全产品的核心页面。** 不是只展示 MBTI 4个字，而是输出综合画像。

#### 4.2.1 页面布局（从上到下）

| 区域 | 内容 |
|------|------|
| **① 大标题** | 你的职业画像：`ENFP · 行动派`（品牌色+大字号） |
| **② 画像卡片** | 一个150字左右的"综合画像" |
| | *例：你是一个行动驱动型的人，习惯边做边想。你关注人的感受，也关注事情的结果。对新事物保持开放，但也需要一定的结构感。* |
| **③ 展板式标签** | 6个标签环形排布： |
| | - 性格倾向：典型ENFP |
| | - 能力倾向：擅长沟通、创意、统筹 |
| | - 兴趣方向：对"人"和"新事物"感兴趣 |
| | - 价值观偏好：追求意义感 > 稳定 |
| | - 职业风险：可能对重复性工作不耐受 |
| | - 综合判断：推荐「创意驱动」方向 |
| **④ 推荐榜单** | 按匹配度排序展示3-5个推荐岗位方向 |
| **⑤ 分享区** | 一键生成海报/截图 |

#### 4.2.2 综合画像的构建逻辑

```
MBTI 结果 → 映射到"能力/兴趣/价值观"三维度
          → 形成综合画像
          → 输出推荐方向（不只看MBTI，综合筛选）

例：
ENFP + 沟通能力强 + 对"人"感兴趣 + 追求意义感
→ 推荐：市场营销（95%）、教育培训（88%）、用户体验（85%）
→ 不推荐：数据统计（40%）、财务分析（35%）
```

**说明：** 这里"能力/兴趣/价值观"不是让用户再填一次，而是基于 MBTI 类型推断。在后续版本中，**用户可以自行补填**这三个维度来校准。

#### 4.2.3 画报模式（Share Card）
- 生成一张"职业画像"卡片（1080×1080px）
- 内容：类型 + 标签 + 推荐 top1 + 一句话速写
- 样式参考：性格色彩图谱 + 渐变色

### 4.3 模块C：岗位推荐与详情

#### 4.3.1 推荐列表
- 基于综合画像，按匹配度排列（支持分页）
- 每条展示：岗位名 / 匹配度 / 简短标签 / 收藏按钮

#### 4.3.2 岗位详情页模板

```
[岗位名称] — [匹配类型]
推荐指数：⭐⭐⭐⭐（基于综合画像）
匹配人格：ENFP / INFP 等

【职业画像匹配说明】
- 这个岗位需要：xxxx（能力）
- 这个岗位提供：xxxx（价值感来源）
- 你的类型在这里会：xxxx（工作状态）

【核心技能清单】
- 硬技能：xxxx、xxxx（附学习资源链接/推荐课程）
- 软技能：xxxx、xxxx

【学习路径（3步）】
第一步（入门）：找到一份相关实习/项目 → 推荐平台/渠道
第二步（积累）：在实战中积累经验 → 推荐做哪些事
第三步（进阶）：形成个人差异化 → 下一步发展路径

*如需了解更具体的学习路径，请填写你的当前阶段（在校/已毕业/在职）*
```

### 4.4 模块D：辅助功能

| 功能 | 说明 |
|------|------|
| **收藏夹** | 收藏岗位/方向，只存本地（localStorage），可导出 |
| **对比** | 2-3个方向横向对比（表格形式：薪资/技能/入行/匹配度） |
| **笔记** | 每个方向下可写"我的感受" |
| **分享** | 生成分享卡片/链接 |
| **再测一次** | 不影响已出结果（保留历史） |
| **自我校准** | 后续版本：补填"能力/兴趣/价值观"来修正推荐 |
| **路径生成** | 用户选了方向后，可生成"我的学习计划"（PDF/笔记） |

### 4.5 模块E：数据与推荐引擎

#### 4.5.1 数据源（V1 可全部前端硬编码）

- **MBTI-岗位映射库**：16型 × 每型3-5个方向（约60-80条）
- **岗位详情**：每个方向含 技能(5-8) / 路径(3步) / 匹配说明
- **画像标签**：每个岗位对应"能力/兴趣/价值观"匹配说明

#### 4.5.2 推荐逻辑（V1 简单规则）

- 一级排序：MBTI 匹配度（高 → 低）
- 二级排序：结合"综合画像"的三维匹配
- 后续：支持用户输入"偏好" → 重新排序

#### 4.5.3 数据量估算

| 数据项 | 数量 |
|-------|------|
| MBTI 类型描述 | 16 条（每型100-200字） |
| 推荐方向 | 16型 × 3-5 = 48-80 条 |
| 岗位详情 | 每个方向配套1条详细 |
| 标签 | 每型 5-8 个个性化标签 |
| 技能 | 每个方向 5-8 项 |
| 学习路径 | 每个方向 3 步 |

## 五、非功能需求

### 5.1 前端
| 需求 | 标准 |
|-----|------|
| 响应式 | 手机端优先（375px—428px），兼容 PC ≥ 1024px |
| 首屏加载 | LCP < 1.5s |
| 结果页动画 | 卡片翻页入场动效（CSS transition） |
| 无用户系统 | 全站无需登录，数据存 localStorage |
| 分享能力 | 原生 Web Share API + 截图（html2canvas） |
| 无后端依赖 | V1 纯静态，V2 再加数据层 |
| 隐私 | 首页标注"不收集任何个人信息" |

### 5.2 合规

- **免责声明**：每页底部标注——"MBTI 及综合画像仅供参考，不代表职业能力。最终选择请结合自身判断。"
- **数据**：测试过程不记录任何个人信息
- **保证**：不提供"结论性"建议（如"你应该选XX"），只提供"参考"
- **防误导**：每型都附带"这个类型可能的弱点"

### 5.3 兼容性
- 主流浏览器 Chrome/Safari/Edge/微信内置浏览器

## 八、风险与假设

| 风险 | 假设 | 应对 |
|------|------|------|
| MBTI 学术争议 | 用户可能质疑"伪科学" | 首页+底部标注"仅供参考"，且后续加入其他维度 |
| 推荐不准 | V1 只有 MBTI 匹配 | V2 开放自填校准，用户可"纠正" |
| 数据量不够 | 60条不足以覆盖所有用户 | V1 标注"加入我们可补充"，V2 开放 UGC |
| 用户不分享 | 核心传播链断 | 可设计"邀请好友做对比"功能 |
