### 1.3 竞品详细分析

#### A. 16Personalities

- **优势**: 全球知名度最高，交互设计优秀，免费模式获客能力强
- **劣势**: 职业规划深度不足，中文内容翻译质量一般
- **关键指标**: 月活 3000 万+，完成率约 65%

#### B. 奥思 MBTI

- **优势**: 中文市场深耕多年，测评 + 课程闭环
- **劣势**: 产品体验老化，过度商业化感知
- **关键指标**: 国内 MBTI 搜索排名靠前

#### C. MBTIonline

- **优势**: MBTI 官方授权，企业市场强势
- **劣势**: 价格门槛高，C 端触达有限
- **关键指标**: B2B 为主营收模式

#### D. 向阳生涯

- **优势**: 一对一深度咨询，结果最精准
- **劣势**: 无法规模化，价格昂贵
- **关键指标**: 高客单价，低用户量

#### E. 简则 MBTI

- **优势**: 轻量快速，社交传播力强
- **劣势**: 科学性不足，职业规划缺失
- **关键指标**: 微信生态传播为主

## 2. 核心差异点分析

### 2.1 我们的核心优势

1. **AI 驱动的个性化职业规划**
   - 不同于竞品的标准化报告，我们通过 AI 深度解读用户测评结果（AI 对话前端与后端通道均已实现，接入真实 LLM Key 后返回真实文案；未配 Key 时降级为占位/提示）。
   - 结合职业数据提供个性化职业建议与匹配度。
   - 支持多轮对话式探索，帮助用户深入理解自我。
2. **测评 + 规划 + 辅导 三位一体**
   - 竞品多为单一功能，我们打通“认识自己 → 规划职业 → 获得成长”完整链路。
3. **免费 + 激活码兑换的可持续模式**
   - 基础测评和报告免费，Pro 套餐 / 1v1 辅导通过激活码兑换实现（后台批量生成，邮件/SMS 触达用户）。
4. **动效化 + 结构化设计**
   - Spring 动画、Reveal 渐显、雷达图等提升体验与完成率。

## 3. PRD 草稿结构

### 3.1 产品定位

- **一句话定位**：一款以 MBTI 人格测评为入口、面向个人成长的职业规划平台，帮助用户「先认识自己，再规划职业」。
- **目标用户**：18-35 岁面临职业选择、转型或成长困惑的年轻人群（在校生、职场新人、转型期从业者）。
- **核心价值**：将「人格测评」与「职业规划」深度打通，输出可执行的职业路线、技能差距分析与个性化学习路径。
- **差异化**：区别于纯测评（16Personalities）与纯咨询（向阳生涯），提供「测评 → 报告 → 职业推荐 → 路线图 → 技能差距 → 学习资源 → 辅导」的完整闭环。

### 3.2 用户故事

> 采用「作为 [角色]，我希望 [目标]，以便 [价值]」的标准格式。

**在校学生 / 应届生**

- 作为一名即将毕业的大学生，我希望通过测评了解自己的人格特质，以便选择更匹配的求职方向。
- 作为一名对专业不满意的在校生，我希望看到与我人格匹配的职业推荐，以便规划考研或转行路径。

**职场新人**

- 作为一名入职一两年的职场新人，我希望明确当前岗位与目标岗位的技能差距，以便有针对性地学习提升。
- 作为一名迷茫的新人，我希望获得个性化学习资源推荐，以便高效补齐能力短板。

**职业转型者**

- 作为一名想转行的从业者，我希望对比多个职业的路线图，以便评估转型成本与可行性。
- 作为一名转型期用户，我希望预约职业规划师进行一对一辅导，以便获得专业建议。

**成长追踪型用户**

- 作为一名长期使用者，我希望多次测评结果能够对比，以便观察自己的成长与变化。
- 作为一名注重进度的用户，我希望在成长仪表盘看到可视化进度，以便保持规划动力。

### 3.4 边界条件与异常处理

**空态处理**

| 场景 | 处理方式 |
| --- | --- |
| 未完成测评即访问报告 | 引导用户先完成测评，展示空态引导页 |
| 无历史测评记录 | 报告对比页展示「暂无可对比记录」空态 |
| 无匹配辅导师 | 辅导匹配页展示空态并推荐通用规划师 |

**异常处理**

| 场景 | 处理方式 |
| --- | --- |
| 接口请求失败 | 展示友好错误提示 + 重试按钮（前端呈现错误态，不使用假数据掩盖失败） |
| 未登录访问受保护页 | 重定向至登录页（RequireAuth 守卫） |
| 未付费访问付费内容 | 重定向至套餐选购页 /pricing（RequirePaid 守卫） |
| 激活码无效/已用/过期 | 展示具体错误原因，引导联系客服 |
| AI 对话超时 | 展示「稍后重试」提示，保留已输入内容 |

**极限边界**

| 场景 | 处理方式 |
| --- | --- |
| 测评中途退出 | 保存作答进度，下次进入可续答 |
| 重复提交测评 | 生成新记录，保留历史记录用于对比 |
| 超长文本输入 | 前端限制输入长度并给出字数提示 |

## 5. AI 能力拓展规划（Roadmap）

> 本章基于当前已落地能力（测评 assessment、纯规则报告 report、AI 对话 ai-chat 三场景/≤50 轮/Agnes 真实 LLM、职业推荐+技能差距+成长计划 career、真人辅导师+排期+实时会话 coaching/realtime、会员/激活码、支付、运营后台 ops）进行延展。**报告本体为刻意"纯规则、无 LLM"设计（成本+合规护城河），AI 能力一律以"旁挂增值层"嫁接，不侵入报告本体。**

### 5.1 分级标准

| 难度 | 判定依据 | 交付周期（参考） |
| --- | --- | --- |
| 🟢 易 | 直接复用 ai-chat / llm-gateway，无需新表、无需新链路，仅新增 prompt 层与 1~2 个接口 | 0.5~1 周 |
| 🟡 中 | 需新增接口 + 少量新表/字段，或改造现有服务（如 career-plan 由规则改 LLM 生成） | 1~2 周 |
| 🔴 难 | 需新链路/新交互/新品类（语音、跨用户、定时推送、可视化），或强外部依赖 | 2~4 周 |

### 5.2 排期总表（按难易递增 + ROI 排序）

| 批次 | 功能项 | 难度 | 嫁接点 | ROI 亮点 | 依赖/风险 |
| --- | --- | --- | --- | --- | --- |
| **P0 第一批** | AI 报告"人话翻译" | 🟢 易 | ai-chat REPORT 场景旁挂摘要 | 免费用户即可体验 AI，提转化 | 与报告本体合规隔离 |
| **P0 第一批** | AI 报告深度问答升级（接入收藏/计划数据做个性化） | 🟢 易 | ai-chat + career 数据注入上下文 | 让"AI 懂你"真正成立 | 上下文 token 控制 |
| **P0 第一批** | AI 追问式测评校准（补 PRD 未落地"自我校准"） | 🟡 中 | assessment + ai-chat | 提升结果精准度与可信度 | 需存追问结果字段 |
| **P1 第二批** | AI 动态成长计划（career-plan 规则改 LLM 生成分周任务） | 🟡 中 | career-plan 服务改造 | 强付费/会员卖点 | 生成质量需人工抽检 |
| **P1 第二批** | AI 咨询前"问题梳理师"（生成咨询提纲给辅导师） | 🟡 中 | coaching 预约链路 + ai-chat | 提高咨询效率与好评、拉高单价 | 与辅导师流程对齐 |
| **P1 第二批** | AI 咨询后"行动纪要"（自动纪要+待办回写成长计划） | 🟡 中 | realtime 消息流 + career-plan | 闭环留存 | 依赖会话消息落库 |
| **P1 第二批** | AI 辅导师智能匹配 | 🟡 中 | coaching + 画像 | 替代人工筛选、提转化 | 匹配规则冷启动 |
| **P2 第三批** | AI 双人/团队协作分析（输入两人 MBTI 出协作建议） | 🟡 中 | 新增独立接口 + ai-chat | 拉新裂变卖点 | 无账号也可体验入口 |
| **P2 第三批** | AI 简历/求职信生成器 | 🟡 中 | 新增品类接口 + ai-chat | 商业化新品类 | 输入表单设计 |
| **P2 第三批** | AI 深度报告"个性化章节"（职业情景推演） | 🟡 中 | 报告旁挂 + ai-chat | 强化 DEEP 付费差异 | 严格与规则报告隔离 |
| **P3 第四批** | AI 模拟面试 + 即时反馈 | 🔴 难 | 新交互（多轮评分） | 高价值付费点 | 评分标准建模 |
| **P3第四批** | AI 面试题库 + 模拟评分 | 🔴 难 | 新品类 + 内容沉淀 | 内容资产沉淀 | 题库冷启动 |
| **P3 第四批** | AI 职业热点日报（个性化推送） | 🔴 难 | scheduler + analytics + 岗位库 | 提留存/复购 | 内容源与推送通道 |
| **P3 第四批** | AI 辅助职业库生产（批量生成岗位描述/职责/技能清单/薪资区间草稿，写 career/career_skill） | 🟡 中 | ops 后台 + llm-gateway + career 表 | 降运营成本、快速扩充职业库 | 仅限权威数据源+AI生成，**严禁爬取招聘平台**；须人工审核后发布 |
| **远期** | AI 简历诊断、AI 职业树可视化、跨会话长期记忆顾问 | 🔴 难 | 多模块整合 | 平台级差异化 | 数据积累与算力 |

### 5.3 落地建议

- **先做 P0 第一批**：三项均嫁接现有 ai-chat/Agnes 通道，改造面小、可快速验证"AI 懂你"卖点，且报告人话翻译能让免费用户提前体验 AI，直接服务付费转化。
- **护城河隔离原则**：所有报告相关 AI 一律"旁挂"，报告本体保持纯规则，确保成本可控与合规不倒退。
- **职业库数据合规红线**：职业库扩充只走「权威数据源（统计局/行业白皮书/正规开放API/官方授权数据集）+ AI 生成草稿 + ops 后台人工审核」路径；**明令禁止爬取 BOSS/猎聘/脉脉等招聘平台**（违反其用户协议/robots，触碰《数据安全法》《个人信息保护法》与数据版权），不得列为产品能力。
- **每项落地路径**：由 PM 先出「PRD + 接口契约（入参/出参/错误码）+ 测试用例」→ 分派后端/DB/前端实施 → tsc 通过 + 真实回归后放行（延续现有分域协同规范）。
