## 四、对外服务契约(WebSocket 事件,不走 HTTP 拦截器)

- **namespace**:`/ws/coaching`;**transports**:`['websocket','polling']`(长轮询降级)。
- **握手鉴权**:`handshake.auth.token` 或 `Authorization: Bearer <token>`;失败下发 `coaching:error`{code:80001} 并断开。
- **上行事件**:
  - `coaching:join` `{ sessionId, lastReceivedSeq? }` → 回 `coaching:joined`{sessionId,room,lastSeq};有遗漏则 `coaching:replay`{fromSeq,messages}
  - `coaching:message` `{ sessionId, clientMsgId, content }` → 房间广播 `coaching:message`{ServerMessage} + 回执 `coaching:ack`{clientMsgId,seq,serverMsgId,ts}
  - `coaching:ack` `{ seq }` → 停止对应下行消息的服务端重发
- **下行事件**:`coaching:joined` / `coaching:message` / `coaching:ack` / `coaching:replay` / `coaching:error`{code,message}
- **可靠性参数**:`ACK_TIMEOUT_MS=5000` / `MAX_RESEND=3` / `MAX_REPLAY=200`。
- **错误码**:`WS_UNAUTHORIZED=80001` / `WS_ROOM_FORBIDDEN=80002` / `WS_SESSION_INVALID=80003`(见 `apps/api/src/common/response.ts`)。

> 已知约束:`scheduleResend` 的重发仅覆盖**本实例**房间内 socket(`roomSockets` 读本实例 adapter),跨实例成员的可靠投递由其**所在实例各自调度**;终极兜底为客户端重连时携带 `lastReceivedSeq` 走断线补发。
