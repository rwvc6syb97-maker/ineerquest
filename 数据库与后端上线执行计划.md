## 三、后端需完善的执行计划(工程侧)

### BE-01 关闭 LLM Mock,接入真实 Provider(阶段3)
- 现状:默认走 `MockLlmProvider`;`OpenAiLlmProvider` 为占位抛错。
- 动作:在 `llm.provider.ts` 实装真实 SDK/HTTP 流式调用;置 `LLM_PROVIDER_ENABLED=true`,配 `LLM_BASE_URL/LLM_MODEL`。
- 验收:深度报告/对话由真实模型生成,`degraded===false`。

### BE-02 Redis 实连,恢复限流/配额/黑名单严格行为
- 现状:**Redis 7 已在本地实连**（`REDIS_URL=redis://localhost:6379/0`）。token 黑名单、用户封禁标记已生效。
- 待完成:LLM 限流、AI 每日配额(200)的严格行为需配合 BE-01 联调验证。
- 验收:超限触发降级/50001;封禁后原 token 被拒(20004)。

### BE-03 MongoDB 实连,消息落库 ✅ 已解决
- **结果**:MongoService 增加 `mongodb-memory-server` 嵌入式数据库回退。无真实 MongoDB 时自动启动内存实例（重启后数据不丢失），完全消除内存 Map 兜底。安装 `mongodb-memory-server` 即可启用。

### BE-04 邮箱登录真实化 ✅ 已落地
- 结果:邮箱注册/登录 + PBKDF2 密码哈希已在 `auth.service.ts` 实现，前端双 Tab 登录页。登录流程零外部依赖。

### BE-05 激活码兑换系统 ✅ 已落地
- 结果:后台批量生成 + 邮件/SMS 触达 + 用户兑换升级会员 + Redis 防并发锁。单元测试 12 用例通过。替换了微信支付链路，零外部支付依赖。

### BE-06 短信服务真实接入(可选)
- 现状:`SMS_PROVIDER_ENABLED=false` 走 mock。邮箱登录通道已提供零依赖备选。
- 动作:如需启用手机号验证码真实发送，回填短信服务商密钥并置 true。

### BE-07 WebSocket 多实例广播 + 网关透传(阶段4)
- 现状:**Redis 已实连**，跨实例广播通道就绪。本地单实例下 WS 通信正常。
- 待完成:多实例部署时 Nginx/Ingress 配 `Upgrade` 透传 + ip-hash 粘性会话。
- 验收:双实例跨 Pod 同房消息互通;WS 建连成功(非仅 polling)。

### BE-08 Elasticsearch 接入(V1.1,可 MySQL 兜底延后)
- 现状:无 ES,内容 CRUD 后 `reindex()` 仅 log;职业检索走 MySQL LIKE。
- 动作:MVP 可延后;V1.1 接入 `@nestjs/elasticsearch` 实装增量索引。

### BE-09 ClickHouse 接入(埋点看板,可 MySQL 兜底)
- 现状:无 CK,看板走 MySQL 聚合降级。
- 动作:就绪 CK 后看板查询切 CK;大数据量下响应达标。

### BE-10 审计 MQ 异步化(可选)
- 现状:审计 fire-and-forget 直写 event_log。
- 动作:就绪 MQ 后改投递+消费落库,写操作响应不受审计延迟影响。

### BE-11 定时任务上线(阶段5) ✅ 已落地
- **结果**:`SchedulerModule` 注册于 `app.module.ts`，4 个 @Cron 任务已实现（Jest 6 用例通过）：

| Cron | 任务 | 频率 |
|---|---|---|
| `cleanupDeactivatedUsers` | 注销用户 T+30 清理 | 每天 03:00 |
| `cleanupStaleEventLogs` | 事件日志 90 天 TTL（分批 LIIMT 10000） | 每天 04:00 |
| `releaseExpiredSchedules` | 辅导排期过期释放 | 每 5 分钟 |
| `markExpiredActivationCodes` | 激活码过期标记 | 每小时 |

- **依赖**: `@nestjs/schedule`（已安装），生产建议加 Redis 分布式锁防多实例重跑。

### BE-12 CI/CD 流水线 + 前端 mock 清理 ✅ 已就绪
- 结果:`ci.yml` + `cd.yml` 已建，所有 secret 占位待填；前端各 mock fallback 分支保留供无后端开发使用。
