## 0. 设计定位与 Design Read

### 0.1 Design Read（一句话设计判断）

> **Reading this as：一款走心的、情感化的人格探索 SaaS 平台，兼顾专业信任感与探索的仪式感，面向 22-35 岁年轻职场人。定位介于 Premium consumer 与 SaaS Landing 之间，采用分屏/滚动叙事的非对称语言，配合有仪式感但克制的动效，以现有蓝橙品牌 token 为骨架，坚决避开 AI 紫/蓝光套路。**

品牌名「InnerQuest 向内求索」释义"内心的远征与探寻"：向内剖析人格（MBTI），向外规划职业（Career）。整个视觉语言应服务于两个动词——**求（探索的主动性）** 与 **索（内省的深度）**。

### 0.2 THREE DIALS 档位设定

依据 skill Section 1，本项目在"Premium consumer / SaaS Landing"之间取值，并针对"测评/报告等信息密集区"做局部下调。

| 档位 | 取值 | 范围含义 | 选型理由 |
|---|---|---|---|
| `DESIGN_VARIANCE` | **7** | 1=完美对称 → 10=艺术混乱 | 打破 AI 默认的"居中 Hero + 三等分卡片"套路，采用分屏（Split Screen）、非对称留白、滚动叙事结构，营造"探索之旅"的空间感；但不到 agency 的 artsy chaos——毕竟是需要信任感的 SaaS，测评/报告页仍需可读、可预期。7 是 Premium consumer landing 的标准值（skill 1.B 预设表）。 |
| `MOTION_INTENSITY` | **5** | 1=静态 → 10=电影级物理 | 需要"仪式感"：Hero 进场、Scroll-Reveal、CTA 物理反馈、测评切题过渡、报告数据揭示——这些动效直接服务于"探索的走心体验"。但测评答题、报告数据这类信息密集区必须克制，避免干扰阅读与决策。5 落在 skill 定义的"Fluid CSS（4-7）"区间，用 `transform`/`opacity` + Spring 缓动即可满足，无需 GSAP scroll-hijack 的重型编排。 |
| `VISUAL_DENSITY` | **4** | 1=艺术画廊 → 10=驾驶舱 | 首屏、营销页、测评引导页留白呼吸（`py-16`~`py-24`），给"内省"以空间；但结果报告、职业匹配等数据区适度增密（更紧凑的栅格、数据卡片、维度条），让"专业分析"有信息厚度。4 是 skill "Daily App（4-7）"区间的低位，兼顾呼吸与信息量。 |

> **动效承诺一致性（skill 5.C "Motion claimed, motion shown"）**：本项目声明 `MOTION_INTENSITY: 5`，则页面必须真实产生动效——Hero 进场、关键区块 Scroll-Reveal、CTA hover 物理反馈至少三项落地；且全部遵守 `prefers-reduced-motion` 降级（见第 9 章）。

## 1. 设计哲学与情绪板

### 1.1 核心情绪：内省 · 探索 · 成长

| 关键词 | 视觉转译 | Token / 实现方向 |
|---|---|---|
| **内省 Introspection** | 沉静、深邃、有留白让人停留思考 | 深蓝夜空基调（`--brand-primary-900/950`），大面积负空间，慢速淡入（`--duration-slow`） |
| **探索 Quest** | 方向感、路径、逐步揭示、非对称的"前行" | 分屏与滚动叙事、Scroll-Reveal Stagger、路径式进度、暖橙作为"指引光点"（`--brand-accent-500`） |
| **成长 Growth** | 温暖、被陪伴、正向反馈、有生命力 | 暖橙点缀、柔和圆角（`--radius-2xl`）、Spring 弹性反馈（`--ease-spring`）、人格族群色的活力表达 |

### 1.2 意象与材质氛围

- **意象**：星图 / 罗盘 / 光的探照 / 内心的地形图。避免使用具象的"大脑""灯泡""齿轮"等 AI 套路化 icon 意象。
- **材质**：以柔和的中性面（`--color-neutral-50`）为主体，深色区域用深蓝而非纯黑（`--color-neutral-950`或 `--brand-primary-950`，禁用 `#000000`，见 skill 9.A）。玻璃拟态仅用于"报告/悬浮结果卡"等少数高价值场景（见第 6 章）。
- **光感**：暖橙作为唯一的"高亮/指引"色，如探索路上的光点，克制使用，绝不做成 AI 蓝光/紫光辉。

### 1.3 反 AI 套路声明（Anti-Default Discipline，skill 0.D）

本项目**明确规避**：AI 紫渐变、深色 mesh 上的居中 Hero、三张等宽功能卡、满屏玻璃拟态、无处不在的无限循环微动画、Inter + slate-900 的默认组合。品牌蓝是 tokens 既定选择（见 2.4 蓝色说明），非 AI 默认反射。

> 全部基于 `design-tokens.css`，**扩展语义映射，不新增冲突色**。遵守 skill 4.2 的 COLOR CONSISTENCY LOCK 与 LILA RULE。

| 角色 | Token | 值 | 用途 |

| 品牌深色 | `--brand-primary-900/950` | `#1e3a8a` / `#101a39` | 深色区块背景、页脚、Hero 暗底 |
| 品牌辅助色 | `--brand-accent-500` | `#f97316` | **唯一强调色**：关键 CTA、进度亮、"探索"指引点 |

| 语义色 | `--color-success/warning/error/info-*` | — | 仅用于状态反馈，不参与装饰 |

### 2.2 强调色锁定（COLOR CONSISTENCY LOCK · 强制）

**全站强调色 = 暖橙 `--brand-accent-500`。** 主 CTA、关键进度、探索指引点、成长正反馈统一使用橙色系。禁止在第 N 屏突然出现一个 teal 徽章或紫色按钮。上线前须逐组件审计：所有"号召行动/高亮"是否统一为橙色系。

- 主 CTA：`--btn-accent-bg`（橙），hover `--btn-accent-hover-bg`，active `--btn-accent-active-bg`
- 次 CTA / 品牌链接：`--brand-primary-500`（蓝，属品牌识别范畴，非第二强调色）
- 二者分工明确：**橙 = 行动，蓝 = 品牌/信息**，不混用。

### 2.3 MBTI 与人格族群色的语义化使用

MBTI 8 维色与 4 族群色是**功能性数据色**，**仅在测评过程与结果报告的数据可视化中出现**，绝不外溢到营销页作装饰（否则违反 COLOR CONSISTENCY LOCK）。

|---|---|---|---|
| E 外向 | `--mbti-ei-energy` | `#f97316` | 活力橙 |
| I 内向 | `--mbti-ei-introvert` | `#3b82f6` | 沉静蓝 |
| S 实感 | `--mbti-sn-sensing` | `#22c55e` | 自然绿 |
| N 直觉 | `--mbti-sn-intuition` | `#8b5cf6` | 灵感紫 |

| J 判断 | `--mbti-jp-judging` | `#0ea5e9` | 秩序青 |
| P 感知 | `--mbti-jp-perceiving` | `#f59e0b` | 随性金 |

**4 族群色（结果类型标识 + 职业匹配分组）：**

| 族群 | Token | 值 |
|---|---|---|
| 分析家 NT | `--personality-analyst` | `#3b82f6` |
| 外交家 NF | `--personality-diplomat` | `#8b5cf6` |

- 维度条（如 E/I 倾向 68%）用对应两端维度色做渐变或分段填充。
- 禁止在同一视图内堆砌 8 种彩色而无信息意义（见 skill 9.C）。

### 2.4 关于蓝色的克制声明（LILA RULE 说明）

skill 4.2 LILA RULE 将"AI 紫/蓝光辉"列为默认禁忌。本项目蓝色为品牌主色，属 tokens 既定、有意为之，**允许保留但须克制执行**：

- **禁止**蓝色外发光/neon glow、蓝色 mesh 渐变背景、蓝色按钮的辉光投影。
- 阴影使用 `--shadow-primary`（蓝色低透明投影，35% alpha）时限于品牌卡片，绝非"发光"。
- 深色区用深蓝 `--brand-primary-950` 作沉稳底，而非"科技蓝光"氛围。
- 全站遵守"一套调色板"：暖中性（`--color-neutral-*`）为骨架，蓝为品牌、橙为强调，不在暖灰与冷灰间摇摆。

## 3. 字体排印系统

> 全部基于 `design-tokens.css` 的字体、字号、字重、行高、字距变量。

### 3.1 字体族分工（引用 token）

| 用途 | Token | 字体栈 | 使用场景 |
|---|---|---|---|
| 展示/大标题 | `--font-family-display` | Plus Jakarta Sans | Hero 主标题、章节大标题、族群名称 |
| 正文/UI | `--font-family-sans` | Inter + PingFang SC | 正文、按钮、表单、导航（中英混排主力） |
| 引文/情感文案 | `--font-family-serif` | Noto Serif SC | 内省性引文、报告开篇寄语（营造"走心"质感） |
| 数据/代码 | `--font-family-mono` | JetBrains Mono | 维度百分比、匹配度数值、编号 |

**排版个性（skill 4.1 Typography as Voice）**：本项目字体不追求"完美平庸"，而通过 **display 字体的紧字距大标题 + serif 引文** 建立情感语气。禁止全站只用 Inter + 一种字重的 AI 默认排版。

### 3.2 字号阶梯与用法

| 层级 | Token | 场景 |
|---|---|---|
| Hero 主标题 | `--font-size-5xl` / `6xl` | 首屏一句话主张 |
| 章节标题 | `--font-size-3xl` / `4xl` | Section H2 |
| 卡片标题 | `--font-size-xl` / `2xl` | 功能卡、报告小节 |
| 正文 | `--font-size-base`（16px）| 段落正文，最小可读 |
| 辅助/标签 | `--font-size-sm` / `xs` | 元信息、标签、脚注 |

**对比强制（skill 4.1）**：相邻层级字号跳跃须明显（至少一个 token 档），杜绝"全是 15-17px 的一片灰"。大标题用 `--font-weight-bold`/`extrabold` + `--letter-spacing-tight`；正文用 `--font-weight-normal` + `--line-height-relaxed`。

### 3.3 中文排版细则

- 中英混排以 `--font-family-sans`（Inter + PingFang SC）为准，中文行高上调至 `--line-height-relaxed`（1.625）保证呼吸。
- 引文/寄语用 serif + `--line-height-loose`，字号 `--font-size-lg`，营造停留感。
- 数字与百分比统一 mono，等宽对齐，避免数据跳动。

## 4. 布局与栅格系统

### 4.1 栅格与容器（引用 token）

- 主栅格 `--grid-columns: 12`，容器最大宽 `--container-max-width: 1280px`。
- 断点：`--breakpoint-xs:375` / `sm:640` / `md:768` / `lg:1024` / `xl:1280` / `2xl:1536`。
- 区块纵向间距用 `--section-gap`，营销区 `py-16`~`py-24`（对应 `--spacing-16`~`--spacing-24`），数据区收紧至 `py-10`~`py-12`。

### 4.2 反居中偏见（ANTI-CENTER BIAS · skill 4.3）

`DESIGN_VARIANCE:7` 要求打破居中默认：

- **首屏**：采用 **Split Screen（左文案右视觉 / 5:7 非对称分栏）**，而非"居中标题 + 居中副标题 + 居中按钮"的 AI 三居中套路。
- **内容区**：交替使用左对齐、非对称留白、内容偏移（如文案占 7 列、视觉占 5 列并 offset）。
- **例外**：单一 CTA 收束区、空状态、加载态可居中——居中是有意的收束，不是默认。

### 4.3 区块布局去重复（Section-Layout-Repetition Ban · skill 4.3）

相邻 Section **禁止连续使用相同布局骨架**。首页节奏建议：

| ��序 | Section | 布局骨架 |
|---|---|---|
| 1 | Hero | Split Screen 非对称分屏 |
| 2 | 核心价值 | Bento 网格（大小不等的 2-3-1 组合） |
| 3 | MBTI 探索介绍 | 左图右文，图偏移出��� |
| 4 | 职业规划能力 | 横向卡片带节奏留白（非三等分） |
| 5 | 用户旅程/流程 | 时间轴 / 路径式纵向叙事 |
| 6 | 社会证明 | 错落卡片墙 |
| 7 | 终局 CTA | 居中收束（唯一允许的强居中） |

### 4.4 Hero 硬规则（skill 4.3 Hero Rules）

- Hero 高度不强制 `100vh`，允许内容自然定高，首屏须露出下方内容边缘以暗示"可探索/可滚动"。
- 禁止"居中大字 + 居中副标题 + 居中双按钮 + 深色 mesh"这一 AI 默认 Hero。
- Hero 视觉侧使用星图/罗盘意象的插画或抽象图形（SVG），非 stock 照片堆叠。

## 5. 材质：圆角、阴影与表面

### 5.1 圆角一致性锁定（SHAPE CONSISTENCY LOCK · skill 4.4）

**全站统一圆角语言 = 柔和圆润**，主用 `--radius-2xl`（卡片）、`--radius-lg`（按钮/输入，`--btn-radius` 已定为 `--radius-lg`）、`--radius-full`（头像/标签/进度端点）。

| 元素 | 圆角 Token |
|---|---|
| 大卡片 / 报告面板 | `--radius-2xl` / `--radius-3xl` |
| 按钮 | `--radius-lg`（`--btn-radius`） |
| 输入框 | `--radius-lg` |
| 标签 / Chip | `--radius-full` |
| 头像 / 族群徽章 | `--radius-full` |
| 图片容器 | `--radius-xl` |

**禁止**：同一视图内混用直角卡片 + 大圆角按钮 + 小圆角输入。圆角是品牌一致性的一部分。

### 5.2 阴影层级（引用 token）

分层克制使用，避免每个元素都投影（skill 4.4 Materiality）：

| 层级 | Token | 用途 |
|---|---|---|
| 微浮 | `--shadow-xs` / `--shadow-sm` | 输入框 focus 前、静态小卡 |
| 常规卡片 | `--shadow-md` | 功能卡、列表项 hover |
| 悬浮/强调 | `--shadow-lg` / `--shadow-xl` |��告悬浮卡、弹窗、下拉 |
| 品牌卡片 | `--shadow-primary` | 蓝色品牌卡（低透明，非发光） |
| 强调 CTA | `--shadow-accent` | 橙色主 CTA hover 抬升 |
| 内嵌 | `--shadow-inner` | 进度槽、输入内凹 |
| 聚焦环 | `--shadow-ring` / `--btn-focus-ring` | 键盘 focus 可见环 |

**阴影 = 用色一致**：卡片投影用中性阴影，仅品牌/CTA 用带色阴影且低透明，绝不做外发光。

### 5.3 玻璃拟态使用边界

仅用于**报告结果的悬浮总览卡** 与 **移动端底部操作条** 两处高价值场景，`backdrop-blur` + 半透明中性白，避免满屏玻璃（AI 套路，skill 9.C）。

## 6. 核心组件材质规范

> 全部对齐 `design-tokens.css` 组件层变量。

### 6.1 按钮（引用 `--btn-*`）

| 变体 | 背景 | 文字 | Hover | Active | 场景 |
|---|---|---|---|---|---|
| Primary（品牌） | `--btn-primary-bg` | `--btn-primary-text` | `--btn-primary-hover-bg` | `--btn-primary-active-bg` | 品牌导航 CTA |
| **Accent（强调）** | `--btn-accent-bg` | 白 | `--btn-accent-hover-bg` | `--btn-accent-active-bg` | **主行动：开始测评、查看报告** |
| Secondary | `--btn-secondary-bg` | `--btn-secondary-text` | `--btn-secondary-hover-bg` | — | 次要操作 |
| Outline | 透明 + `--btn-outline-border` | `--btn-outline-text` | `--btn-outline-hover-bg` | — | 并列次选 |
| Ghost | 透明 | `--btn-ghost-text` | `--btn-ghost-hover-bg` | — | 工具栏、卡内操作 |
| Danger | `--btn-danger-bg` | 白 | `--btn-danger-hover-bg` | — | 删除/退出测评 |

- 尺寸：`--btn-height-xs`~`--btn-height-xl`，圆角 `--btn-radius`。
- 焦点：键盘 focus 必须显示 `--btn-focus-ring`（见第 9 章无障碍）。
- Tailwind 示例（Accent 主 CTA）：
  ```html
  <button class="inline-flex items-center justify-center rounded-lg h-12 px-6
    bg-[var(--btn-accent-bg)] text-white font-semibold
    shadow-[var(--shadow-accent)] transition-transform
    hover:bg-[var(--btn-accent-hover-bg)] hover:-translate-y-0.5
    active:bg-[var(--btn-accent-active-bg)] active:translate-y-0
    focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-offset-2">
    <i data-lucide="compass" class="mr-2 h-5 w-5"></i>
    开始探索我的人格
  </button>
  ```

### 6.2 输入框（引用 `--input-*`）

- 高度 `--input-height`，内距 `--input-padding`，边框 `--input-border-*`，圆角 `--radius-lg`。
- 状态：默认 `--input-border`，focus `--input-border-focus` + `--shadow-ring`，error `--color-error-*`。

### 6.3 卡片

- 底 `--card-bg`（`--color-neutral-50/白`），圆角 `--radius-2xl`，常规阴影 `--shadow-md`，hover 抬升至 `--shadow-lg` + `translateY(-2px)`。
- 报告族群卡：以用户族群色作左边 4px 强调条或顶部细渐变，主体仍中性面。

## 7. 交互状态规范

> 每个交互元素必须定义 4 态（skill 4.5 States as First-Class）：默认 / hover / active / focus，另按需补充 disabled / loading。

### 7.1 状态矩阵（通用）

| 状态 | 视觉变化 | Token / 时长 |
|---|---|---|
| Default | 基础样式 | — |
| Hover | 抬升 `translateY(-2px)` + 阴影升一级 + 背景加深 | `--transition-transform`（`--duration-fast`） |
| Active | 回落 `translateY(0)` + 轻微 `scale(0.98)` | `--duration-instant` |
| Focus（键盘） | 显示 `--shadow-ring` / `--btn-focus-ring` | 立即，无过渡 |
| Disabled | `opacity: 0.5` + `cursor: not-allowed` | 无 hover 反馈 |
| Loading | 内容替换为 spinner，禁用交互 | 见 7.3 |

### 7.2 关键场景状态

- **测评选项卡**：hover 边框转对应维度色浅调；选中态填充维度色 8% 透明 + 边框维度色 + 左侧对勾（`--ease-spring` 弹入）。
- **报告维度条**：进入视口后从 0 填充至目标百分比（见 8.3 数据揭示动效）。
- **导航链接**：hover 下方橙色下划线由中间向两侧展开（`--duration-fast`）。

### 7.3 加载与空状态

- 加载：骨架屏（skeleton）用 `--color-neutral-100`~`200` 之间的柔和 shimmer，避免旋转 spinner 满屏。测评提交用按钮内嵌 loading。
- 空状态：居中收束（本处允许居中），配轻插画 + 一句引导文案 + 单一 Accent CTA，不堆砌。

## 8. 动效规范（MOTION_INTENSITY: 5）

> 定位"Fluid CSS（4-7）"：以 `transform`/`opacity` + Spring 缓动实现有仪式感但克制的动效。**信息密集区（测评答题、报告数据）必须克制**，动效只服务于理解与愉悦，不干扰决策。

### 8.1 动效时长与缓动（引用 token）

| Token | 值 | 用途 |
|---|---|---|
| `--duration-instant` | 极短 | active 回弹、即时反馈 |
| `--duration-fast` | 快 | hover、下划线、小元素过渡 |
| `--duration-normal` | 中 | 卡片进场、切换 |
| `--duration-slow` | 慢 | Hero 进场、大区块淡入 |
| `--ease-spring` | `cubic-bezier(0.34,1.56,0.64,1)` | 有回弹的进场/选中（主力缓动） |
| `--ease-bounce` | — | 强调性弹跳（庆祝、报告揭晓） |
| `--transition-*` | — | 组合过渡快捷变量 |

**统一缓动语言**：全站进场/交互默认用 `--ease-spring`，营造温暖弹性；线性/匀速仅用于进度条与 shimmer。

### 8.2 Spring Physics 参数（若使用 JS 动画库 / Framer Motion 语义）

即便以 CSS 为主，如需 JS 编排（React + Framer Motion），统一采用如下 spring 语义，与 `--ease-spring` 观感对齐：

```js
// 全局 Spring 预设（与 --ease-spring 观感一致）
export const springPresets = {
  // 进场：卡片、区块揭示
  gentle:  { type: "spring", stiffness: 120, damping: 20, mass: 1 },
  // 交互反馈：CTA、选中态
  snappy:  { type: "spring", stiffness: 300, damping: 24, mass: 0.8 },
  // 强调揭晓：报告结果、族群揭示
  bouncy:  { type: "spring", stiffness: 200, damping: 12, mass: 1 },
};
```

### 8.3 Canonical 动效模式（skill 5.B）

**（1）Hero 进场 - 分层淡入上移（Reveal-Stagger）**

```css
@keyframes fadeUp {
  from { opacity: 0; transform: translateY(24px); }
  to   { opacity: 1; transform: translateY(0); }
}
.hero-item {
  opacity: 0;
  animation: fadeUp var(--duration-slow) var(--ease-spring) forwards;
}
.hero-item:nth-child(1) { animation-delay: 0ms; }
.hero-item:nth-child(2) { animation-delay: 80ms; }
.hero-item:nth-child(3) { animation-delay: 160ms; }
.hero-item:nth-child(4) { animation-delay: 240ms; }
```

**（2）Scroll-Reveal Stagger（区块滚动揭示 - IntersectionObserver 驱动）**

> 严禁使用 `window.addEventListener('scroll')` 逐帧追踪滚动值（skill 5.D Forbidden Patterns）。必须用 IntersectionObserver 触发一次性 class 切换。

```js
// 正确做法：观察进入视口，添加一次性揭示 class
const io = new IntersectionObserver((entries) => {
  entries.forEach((entry) => {
    if (entry.isIntersecting) {
      entry.target.classList.add('is-revealed');
      io.unobserve(entry.target); // 只触发一次
    }
  });
}, { threshold: 0.15, rootMargin: '0px 0px -10% 0px' });

document.querySelectorAll('[data-reveal]').forEach((el) => io.observe(el));
```

```css
[data-reveal] {
  opacity: 0;
  transform: translateY(32px);
  transition: opacity var(--duration-slow) var(--ease-spring),
              transform var(--duration-slow) var(--ease-spring);
}
[data-reveal].is-revealed { opacity: 1; transform: translateY(0); }
/* 子元素错峰 */
[data-reveal] > * { transition-delay: calc(var(--i, 0) * 80ms); }
```

**（3）报告维度条数据揭示（进入视口后填充）**

```css
.dimension-bar__fill {
  width: 0;
  transition: width var(--duration-slow) var(--ease-spring);
}
.dimension-bar.is-revealed .dimension-bar__fill {
  width: var(--value); /* 如 68% */
}
```

```js
// 复用上方 IntersectionObserver，命中时读取 data-value 设为 CSS 变量
entry.target.style.setProperty('--value', entry.target.dataset.value + '%');
entry.target.classList.add('is-revealed');
```

**（4）CTA 物理反馈（hover 抬升 + active 回落）**

```css
.cta {
  transition: transform var(--duration-fast) var(--ease-spring),
              box-shadow var(--duration-fast) ease;
  will-change: transform;
}
.cta:hover  { transform: translateY(-2px); box-shadow: var(--shadow-accent); }
.cta:active { transform: translateY(0) scale(0.98); }
```

### 8.4 禁止的动效模式（skill 5.D · 强制）

- **禁止** `window.addEventListener('scroll', ...)` 逐帧读取滚动位置驱动动画（用 IntersectionObserver 或 CSS `animation-timeline`）。
- **禁止** 用 `useState`/state 追踪连续变化值（如滚动进度、鼠标坐标）触发 re-render 做动画（用 CSS 变量或 ref 直写）。
- **禁止**无限循环的装饰性微动画（呼吸光点、常驻抖动）——除非是明确的加载态。
- **禁止**过场 scroll-hijack、超过 1 屏的强制视差劫持（超出 MOTION:5 范畴）。
- 动画一律作用于 `transform` / `opacity`（GPU 合成），禁止动画 `width`/`height`/`top`/`left`（维度条 width 属受控数据揭示，例外且低频）。

### 8.5 信息密集区克制原则

测评答题页、报告数据页遵守**低动效**：仅保留选项选中的 `--ease-spring` 微反馈与维度条一次性填充，切题用简单 `--duration-fast` 淡入淡出，**不做**花哨转场，保证专注与效率。

## 9. 性能、可访问性与暗色模式护栏

### 9.1 Core Web Vitals 目标（skill 6）

| 指标 | 目标 | 保障措施 |
|---|---|---|
| LCP | < 2.5s | Hero 首图优先加载、字体 `font-display: swap`、关键 CSS 内联 |
| INP | < 200ms | 交互反馈用 CSS transform（不触发布局）、避免主线程长任务 |
| CLS | < 0.1 | 图片/容器预留宽高比、字体回退度量匹配、避免动画位移已固定内容 |

### 9.2 硬件加速与克制

- 动画只用 `transform`/`opacity`，必要时 `will-change: transform`（用后移除，避免层爆炸）。
- z-index 严格分级，全部引用 token：`--z-index-sticky:1020` / `--z-index-modal:1050` / `--z-index-toast:1080` / `--z-index-loading`。禁止随手写 `z-index: 9999`。

### 9.3 无障碍（WCAG）

- **对比度**：正文对背景 ≥ 4.5:1，大字 ≥ 3:1。橙色 CTA 上文字用白色须验证达标；橙色小字禁止置于浅底。
- **键盘可达**：所有交互元素可 Tab 聚焦，focus 显示 `--btn-focus-ring` / `--shadow-ring`，不移除 outline 而不提供替代。
- **语义化**：使用 `main` / `header` / `nav` / `section` / `article`，标题层级 h1→h2→h3 不跳级；图标按钮加 `aria-label`。
- **色彩非唯一信息载体**：维度倾向除颜色外须有文字标签与百分比（色盲友好）。
- 测评进度用 `role="progressbar"` + `aria-valuenow`。

### 9.4 prefers-reduced-motion（强制降级 · skill 6）

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
  [data-reveal] { opacity: 1 !important; transform: none !important; }
  .dimension-bar__fill { transition: none !important; }
}
```

所有揭示类动效在降级下必须**直接显示终态**（��容可见、数据可读），绝不因关闭动效而丢失信息。

### 9.5 暗色模式协议（skill 8）

- 深色底用深蓝 `--brand-primary-950` / `--color-neutral-950`，**禁用纯黑 `#000000`**。
- 暗色下文字用 `--color-neutral-100/200`（非纯白），降低眩光。
- 阴影在暗色中减弱，改用边框（`--color-neutral-800`）与轻微亮度分层表达层级。
- 品牌橙在暗底保持可用，蓝色降饱和避免刺眼；族群色在暗色下统一提亮一档保持辨识。
- 暗色非简单反色：重新校准每一层表面亮度，保持"沉静内省"而非"冰冷科技"。

## 11. AI TELLS 禁忌清单（skill 9 · 上线前逐项审计）

> 以下为"�眼 AI 生成"的破绽，本项目**零容忍**。上线前须逐条 checklist 核对。

### 11.A 内容真实性

- 禁止 "Jane Doe / John Smith / Acme Inc / Lorem ipsum" 等占位内容遗留到成品。
- 禁止假截图 div（用彩块假装产品界面），用真实组件或明确标注的示意图。
- 用户评价、数据须为真实或明确标注为"示例"。

### 11.B 排版破绽

- **EM-DASH 零容忍**：全站禁用 `—`（em-dash）与 `–`（en-dash），一律用普通连字符 `-` 或重写句子。本文档自身已全程遵守。
- 禁止 eyebrow 标签（大标题上方的全大写小字 "OUR FEATURES"）滥用，最多克制出现一处。
- 禁止全站单一字重的"一片灰"排版（见第 3 章对比强制）。

### 11.C 视觉套路

- 禁止 AI 紫渐变、蓝/紫 neon glow、深色 mesh 背景（见 2.4）。
- 禁止满屏玻璃拟态（限第 5.3 两处）。
- 禁止装饰性状态点（无意义的常驻小圆点 / 呼吸灯）。
- 禁止三张等宽功能卡的默认布局（用 Bento 或节奏留白）。
- 禁止无处不在的居中（见 4.2 ANTI-CENTER BIAS）。

### 11.D 动效套路

- 禁止无限循环装饰微动画、`scroll` 事件逐帧动画、连续值 state 驱动动画（见 8.4）。
- 声明了 `MOTION:5` 就必须真有动效落地（见 0.2 动效承诺一致性）。

### 11.E 色彩一致性

- 上线前逐组件审计强调色是否统一为橙色系（COLOR CONSISTENCY LOCK，见 2.2）。
- MBTI/族群色不得外溢到营销页作装饰（见 2.3）。

### 11.F 圆角/材质一致性

- 逐视图审计圆角是否统一（SHAPE CONSISTENCY LOCK，见 5.1）。
- 阴影不得做成外发光，带色阴影仅限品牌/CTA（见 5.2）。
